# Zoomy (ZoomTransition) — 설계 문서

> 2026-07-03. 초안 → 3방향 적대적 리뷰(UIKit 기술 검증 / API·DX 공격 / 시나리오 공격) 반영 v2.
> 저장소 폴더: `~/Desktop/ZoomTransition` · Swift 모듈명: `Zoomy` (모듈명==핵심 타입명 충돌 회피)

## Context

iOS 홈 화면에서 앱 아이콘을 탭하면 앱이 아이콘 위치에서 확대되며 열리고, 닫으면 다시 아이콘으로 축소되는 zoom transition을 일반화한 Swift UIKit 오픈소스 라이브러리. iOS 18 `UIViewController.Transition.zoom(sourceViewProvider:)`과 같은 효과를 **iOS 15+에서, 더 커스터마이즈 가능하게** 제공. 그린필드 프로젝트.

### 확정 사항
- **지원 범위**: `UINavigationController` push/pop **및** modal present/dismiss 둘 다
- **이름**: 모듈 `Zoomy`, 타입 프리픽스 `Zoom`
- **최소 버전**: **iOS 15 (사용자 확정)**, Swift 5.9+, UIKit 전용, 의존성 0, SPM 배포
  - 버전 영향 검토: 핵심 전환 API와 sharp edge 워크어라운드는 iOS 15~18에서도 동일 — 구조 변경 없음. `prefersCrossFadeTransitions`(iOS 14+)를 availability 게이트 없이 사용 가능해지는 단순화만 발생. 내비바 고스팅 워크어라운드는 iOS 15가 지원 범위에 남으므로 유지.
- **인터랙티브 dismiss**: 팬 제스처 finger-follow 축소, 스크럽/취소/재잡기 (interruptible)

## 아키텍처 개요 — 2계층 분리

- **Engine** (`ZoomTransition` + internal 타입들): UIKit 델리게이트 어댑터, 프레젠테이션 컨트롤러, 소스 뷰 재해석, 제스처, `UIViewPropertyAnimator` 생명주기. 픽셀은 그리지 않음.
- **Strategy** (`ZoomTransitionAnimating`): 픽셀 담당. 플래그십 `ZoomAnimator`, 폴백 `CrossDissolveAnimator`. **v1에서는 internal** — 인터랙티브 엔진의 계약(단일 애니메이터 규칙, 2-애니메이터 분리, 스크럽 안전성)은 타입 시스템으로 강제 불가능하고 엔진이 실사용을 거치기 전에 공개하면 semver 함정. 파일 구조는 공개 전제로 유지, 1.x에서 승격.

v1 공개 타입 ~5개, 나머지 전부 `internal`.

### 소비자 사용법

```swift
// modal — 안정 ID로 조회 (indexPath 캡처는 데이터 변경 시 오동작하므로 예제부터 ID 기반)
let detail = PhotoDetailViewController(photo: photo)
detail.zoomTransition = ZoomTransition { [weak self] _ in
    guard let ip = self?.dataSource.indexPath(for: photo.id) else { return nil }
    return (self?.collectionView.cellForItem(at: ip) as? PhotoCell)?.photoView
}
present(detail, animated: true)

// push — nav 설정 시 1회 (핸들 보관)
let zoomDelegate = navigationController.enableZoomTransitions()  // -> ZoomNavigationDelegate
```

### API 형태: 클로저 `sourceViewProvider` (iOS 18 `.zoom`과 1:1)

- 클로저가 **모든 애니메이션 단계 시작마다 재호출** → 셀 재사용·스크롤 아웃을 구조적으로 처리. `nil` 반환 → 폴백.
- **정직한 한계 명시**: 재해석이 해결하는 것은 *셀 정체성*이지 *데이터 정체성*이 아님 — 아이템 삭제/이동 매핑은 소비자 몫(위 예제처럼 ID→indexPath 조회). README에 명시.
- **클로저 계약** (doc comment로 명문화): 메인 스레드 호출, 순수해야 함(레이아웃 변형·내비게이션 금지), 반환 뷰는 non-transitioning 쪽 계층 소속(목적지 자신의 서브뷰 금지 — debug assert `!isDescendant(of: zoomedVC.view)`), 위반 시 debug assert + 폴백.
- 기각: 소스 VC 프로토콜 채택(목적지→셀 매핑 부기 전가), Hero식 문자열 ID(stringly-typed, 재사용 셀에서 최취약).

### Public API 스케치

```swift
@MainActor
public final class ZoomTransition: NSObject {
    public typealias SourceViewProvider = (Context) -> UIView?
    /// 파라미터 레이블 iOS 18과 1:1: sourceViewProvider
    public init(configuration: Configuration = .default,
                sourceViewProvider: @escaping SourceViewProvider)
    public let configuration: Configuration        // 불변 — 전환별 스냅샷 의미론 논쟁 원천 차단
    public weak var delegate: ZoomTransitionDelegate?
    /// 설치 후 접근 가능. 스크롤뷰 등과의 제스처 중재를 소비자가 직접 배선하는 유일한 통로.
    public var dismissalPanGesture: UIPanGestureRecognizer? { get }

    public struct Context {
        public weak var zoomedViewController: UIViewController?   // weak — 캡처 순환 억제
        public weak var sourceViewController: UIViewController?
        public let phase: Phase          // appearing / disappearing
        public let operation: Operation  // push / pop / present / dismiss
        public let isInteractive: Bool
    }
}

@MainActor
public protocol ZoomTransitionDelegate: AnyObject {  // 전부 기본 구현
    func zoomTransition(_ t: ZoomTransition, willBegin context: ZoomTransition.Context)
    func zoomTransition(_ t: ZoomTransition, didEnd context: ZoomTransition.Context,
                        result: ZoomTransition.Result)
}
// Result: completed/cancelled × fallbackReason(옵셔널) — 계측 + "왜 zoom이 안 돌았나" 디버깅 통로.
// FallbackReason: .sourceUnresolved | .notWired | .reentrant | .reduceMotion | .unsupportedOperation
// DEBUG에서는 os_log 병행.

extension UIViewController {
    /// present/push 전에 할당. setter는 associated object 저장 + 이전 modalPresentationStyle/
    /// transitioningDelegate 스냅샷 후 설치. nil 할당 시 스냅샷 원값 복원(임의 상수 복원 금지).
    /// precondition(presentingViewController == nil, "present 이후 할당 금지")
    public var zoomTransition: ZoomTransition? { get set }
}

extension UINavigationController {
    /// 프록시 설치의 유일한 편의 API. Bool 아님 — 상태를 거짓말하는 getter 문제 회피.
    /// RxCocoa 등 delegate-proxy 라이브러리와 병용 금지 (README 명시) — 그 경우 아래 명시적 API 사용.
    @discardableResult
    public func enableZoomTransitions() -> ZoomNavigationDelegate
}

/// 자기 delegate를 소유하는 앱(Coordinator/RxCocoa)의 **1순위 API**
@MainActor
public final class ZoomNavigationDelegate: NSObject, UINavigationControllerDelegate {
    public init(forwardingTo downstream: UINavigationControllerDelegate? = nil)
    /// didSet에서 nav.delegate 재할당으로 UIKit의 delegate capability-flag 캐시 무효화
    public weak var downstream: UINavigationControllerDelegate? { get set }
}
```

**Configuration** (불변, YAGNI):
`spring(response: 0.44, dampingRatio: 0.85)` — `UISpringTimingParameters`에 response init이 없으므로 `stiffness=(2π/response)², damping=2·ratio·√stiffness`로 유도 · `dimmingColor`(nilable) · `cornerMorph(.automatic/.fixed/.none)` · `interactiveDismissal(.pan/.disabled)` · `fallback(.crossDissolve/.systemDefault)` · `respectsReduceMotion = true` · `resignsFirstResponders = true`

**단일 부착 규칙**: 하나의 `ZoomTransition` 인스턴스를 두 VC에 할당하면 debug assert ("VC당 1개 생성"). 공유 상태 머신이 남의 전환을 직렬화하는 비결정성 차단.

### UIKit 훅

- **Modal**: `.custom` + internal 어댑터. `ZoomPresentationController`(`shouldRemovePresentersView = false`)가 디밍 소유. **README 최상단 캐비앗**: `.custom`은 presenter의 `viewWillAppear/Disappear`를 호출하지 않음 — `.fullScreen`에서 이전해 오는 앱의 분석/리프레시 로직이 조용히 죽는 지점.
- **Navigation 프록시**: `animationControllerFor`/`interactionControllerFor`만 직접 구현. **zoom을 vend하는 조건은 인접 단일 스텝 push/pop뿐** — `popToRootViewController`·back버튼 롱프레스 다단 pop·`setViewControllers`는 nil 반환(시스템 전환) + `.fellBack(.unsupportedOperation)`. pop은 `toVC`가 push 시점에 `fromVC` 바로 아래 VC일 때만.
- **포워딩 안전망**: `responds(to:)` + `forwardingTarget(for:)`에 더해 `methodSignature(for:)` + `forwardInvocation:` no-op 폴백 필수 — downstream이 죽은 뒤 UIKit의 캐시된 capability flag가 stale 호출을 보내는 `doesNotRecognizeSelector` 크래시 방지. 프록시 체인 순환 가드(`chainContains(self)` assert). 스위즐링/KVO 없음.
- **시스템 edge-swipe pop — 명시적 배선 필요** (초안의 "공짜" 주장은 오류): 커스텀 animator가 vend되면 시스템 `interactivePopGestureRecognizer`는 커스텀 전환을 구동하지 않음(죽거나 nav 상태 파손). 해법: 해당 recognizer에 **자체 target/action 추가** + `.began`에서 직접 `popViewController(animated: true)` 호출 + "gesture-initiated" 플래그 → 드라이버가 `wantsInteractiveStart = true`로 개시, translation을 zoom progress로 변환. zoom 미적용 VC에서는 스톡 동작 보존(원 delegate 위임).

## 내부 엔진 핵심 결정

### 전환 해부

- **PortalView**: `clipsToBounds` + `layer.cornerRadius`(+`.continuous`) — UIView-animatable이라 스크럽/리버스 가능(mask 레이어 회피 근거). 단 스크럽 중 radius를 읽을 땐 **model이 아닌 `layer.presentation()`** 값 사용.
- **라이브 뷰 정책**: 목적지는 라이브 뷰 reparent + transform 애니메이션. placard는 `snapshotView(afterScreenUpdates: false)` — **nil/blank 허용 설계**: placard 없이도 rect만으로 zoom 성립, nil이면 생략하고 진행(크래시·강제 언랩 금지).
- **Safe area churn 대응**: 스케일된 portal 안의 라이브 뷰는 비행 중 `safeAreaInsets`가 요동쳐 재레이아웃 점프 유발 → 비행 중 zoomed VC에 `additionalSafeAreaInsets`로 **최종 인셋 강제 주입**, cleanup에서 제거. `ZoomPresentationController.containerViewWillLayoutSubviews`에 `isTransitioning` 가드(전환 중 frame 스탬핑 금지) + 회전 시 presenter push-back transform은 identity 리셋→frame 적용→재적용 순서.
- **Zoom-in**: toView를 finalFrame으로 선레이아웃 → portal 확대 + cornerRadius 모핑 + placard 앞 40% 페이드(keyframes) + 디밍 + presenter push-back. 소스 `isHidden = true`는 **독립 복원 토큰 객체**에 기록 — 이 객체의 deinit 자체가 복원을 수행해, UIKit이 컨텍스트를 버리는 비정상 경로에서도 셀 실종·탭바 alpha 0 잔존 방지.
- **소스 해석 강화** (양방향 동일 검증 — appearing에도 적용): non-nil · `window != nil` · non-transitioning 쪽 계층 소속 · rect 교차 + **조상 hidden/alpha≈0 워크** + **가시 영역 계산**(clipsToBounds 조상 체인으로 visible rect 산출, `visible/full < 0.35`면 폴백; 통과 시 portal 최종 상태에 클립 반영 — 반쯤 가린 셀로 착지할 때 "유령 콘텐츠" 방지) + 셀 하이라이트 애니메이션 중이면 **presentation layer 기하** 사용. `scrollToItem` 복원 패턴은 `animated: false` + `layoutIfNeeded()`로 문서화(같은 틱 blank 스냅샷 방지).

### 인터랙티브 dismiss 엔진

1. **`wantsInteractiveStart`는 동적**: 제스처(팬/엣지 스와이프)가 이미 활성이면 `true`(첫 프레임 점프 방지 — Apple 샘플과 동일), 탭 개시 전환은 `false`로 vend해 두고 mid-flight grab 시 `pauseInteractiveTransition()` (WWDC 2017 패턴은 이 경로 전용).
2. **커스텀 `UIViewControllerInteractiveTransitioning`** (percent-driven 회피): finger-follow 기하 + 스프링 조합에 필수. `update/finish/cancel`을 직접 호출.
3. **2-애니메이터 분리 + 완료 순서**: transition animator(대칭 속성 전용)는 settle 동안 **paused 유지**(재잡기 보존). geometry 스프링(velocity 반영 fresh 생성)과의 완료는 **배리어**로 묶어 마지막 완료가 `completeTransition`을 발화. "transition animator 완료가 곧 completeTransition"(초안)은 스프링 비행 중 컨테이너 철거 사고를 유발하므로 폐기. `settling → idle` 전이 조건 = **모든 live animator 완료** (`ActiveTransition`이 animator 배열 보유).

- **Progress 정의 (radial)**: 수직-이동량 progress는 2D follow와 불일치(가로로 카드를 던지면 progress≈0 → 내비바 동결·취소 오판) → progress = **방향 가중 radial displacement**, 완료 판정도 2D velocity를 변위 방향에 투영해 계산. `velocity 투영 > 500` 즉시 dismiss / 역방향 플릭 취소 / 그 외 projection > 0.5.
- **`fractionComplete` 0.995 클램프**(paused animator 1.0 도달 시 하위 CAAnimation 제거로 역스크럽 사망 — 실재 sharp edge), 취소는 `isReversed = true` **먼저** → `continueAnimation(timingParameters: nil, ...)` (non-nil 공급 시 reversed 무시되는 UIKit 버그 회피).
- **취소-정착 중 재잡기**: reversed animator는 **절대 스크럽 금지** — pause → `stopAnimation(false)` + `finishAnimation(at: .current)`로 presentation 값을 model에 스탬핑 → 현재 값 기준 **fresh animator** 재생성. `f_effective = isReversed ? 1-f : f` 매핑은 pure 상태 머신에 넣고 테이블 테스트.
- **스크롤뷰 공존**: 팬 시작 조건 = 하향 지배적 translation ∧ 목적지 스크롤뷰가 top에서 정지. 그 외 케이스는 공개된 `dismissalPanGesture`로 소비자가 `require(toFail:)` 배선.
- **키보드**: present 시 `presentingView.endEditing(true)`, dismiss 제스처 `.began`에서 `zoomedVC.view.endEditing(true)` (키보드는 별도 윈도우라 portal과 함께 축소되지 않음 — `resignsFirstResponders` 컨피그).

### 상태 머신 (pure Swift)

```
idle → animating → interactive(잡기) → settling(놓기) → idle
         ↑______________ re-grab ______________|
비정상: sizeWillChange / scene-background → fast-forward → idle
```
불법 전이 debug assert, 사이드이펙트는 데이터로 반환. 재진입: `state != .idle`이면 begin 거부 + `.fellBack(.reentrant)`. provider 호출은 begin **도중**이므로 provider 내 내비게이션 호출은 계약 위반(assert).

### 접근성 (엔진 레벨 — Strategy가 잊을 수 없는 위치)

- **Reduce Motion / Prefer Cross-Fade**: animator 선택 시점에 `UIAccessibility.isReduceMotionEnabled` / `prefersCrossFadeTransitions` 검사(iOS 15 min이라 availability 게이트 불필요) → `CrossDissolveAnimator`로 단락, `.fellBack(.reduceMotion)` 보고. `respectsReduceMotion = true` 기본.
- **VoiceOver**: 구동 중이면 비인터랙티브 크로스페이드(비행 중 reparent가 포커스 드랍). `ZoomPresentationController`가 `accessibilityViewIsModal` 설정(presenter 뷰가 계층에 남아 있어 필수), cleanup에서 `.screenChanged` post, `accessibilityPerformEscape` → dismiss.

### 엣지케이스 처리 요약

| 케이스 | 전략 |
|---|---|
| dismiss 시 소스 소멸/가림/hidden 조상 | 강화된 검증(위) → 폴백 `.crossDissolve` + `.fellBack(.sourceUnresolved)` |
| 소스 이동(회전/reload) | 기하 무캐시 — begin·settle 재해석. `.fullScreen` 변형은 해석 전 `presenterView` 컨테이너 재삽입 + `layoutIfNeeded()` 강제(회전 후 stale 기하 방지) |
| 전환 중 회전/사이즈 변경 | fast-forward — **`ActiveTransition`의 모든 live animator** 순회 마감, interactive 중이면 progress>0.5 기준 방향 결정, 팬 recognizer `isEnabled` 토글로 플러시, 종료 프레임은 새 bounds로 재계산 |
| 백그라운드 | **`UIScene.didEnterBackgroundNotification`을 windowScene 필터로** 관찰(앱 스코프는 iPad 멀티윈도우에서 미발화) → fast-forward |
| 전환 중 프로그램적 내비게이션 | 복원 토큰 객체 deinit 백스톱 + `animationEnded`에도 fast-forward 배선 |
| 탭바/`hidesBottomBarWhenPushed` | 바 스냅샷 페이드, 실제 바 alpha 복원은 토큰 객체 |
| 내비바 | **기본 권장: detail에서 바 숨김(사실상 필수)** — mid-flight grab 시 UIKit 바 크로스페이드 되감기 신뢰 불가(iOS 13–15 고스팅). 양쪽 표시+large title은 바 스냅샷 전략, 취소된 인터랙티브 pop 후 `navigationBar.setNeedsLayout()` 강제 |
| status bar | modal: `modalPresentationCapturesStatusBarAppearance = true`(.custom에 유효) + 애니메이션 블록 내 update, **취소 시 블록 밖 재-update**. push: nav가 topVC에 위임 안 하는 UIKit 제약 — `childForStatusBarStyle` 오버라이드를 README 캐비앗 + Example 시연 |
| presentation style | `.custom`(기본)/`.fullScreen`/`.overFullScreen` 지원, sheet/popover는 debug assert. **sheet 안에서 구동되는 nav**: cornerMorph `.automatic`은 "컨테이너가 풀스크린 윈도우를 채울 때만 디바이스 코너, 아니면 0/호스트 radius" 휴리스틱 (`UIScreen.main` 사용 금지, window 기준) |
| 메모리 | 목적지 VC가 associated object로 retain. 전환별 강참조는 `ActiveTransition` 단일 구조체, cleanup에서만 nil. provider `[weak]` 캡처 README 강조 |

**단일 출구 cleanup**: 멱등, 복원 토큰 실행 → transform=identity·원위치 reparent → scaffolding 제거 → `.screenChanged` post → `completeTransition` 직전 실행. 호출 지점: 최종 animator 완료 / `animationEnded` / fast-forward / deinit. `finishAnimation`은 `.inactive` 상태 가드 필수(예외 던짐).
